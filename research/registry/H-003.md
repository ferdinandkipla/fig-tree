# H-003: Time-Exit Structure — Truncating Losers Early Adds Value

## Prior evidence (recorded before test)
- Phase 1 USDJPY: time exits +$56.31/trade (n=68) vs. stop losses
  -$98.75/trade (n=113).
- Direction replicated in M0 regenerated logs across instruments.
- M2: exits are the suspected locus of structure (entries shown
  worse-than-random; H-001 killed the entry-depth hypothesis on TRAIN).

## The survivorship trap (why the naive test is invalid)
"Time exits have positive expectancy" is NOT the hypothesis. A trade
reaches the time exit only by surviving N bars without hitting the stop
-- conditioning on exit_reason == time_exit selects on the outcome
itself. Any strategy, including the null, shows better P&L on
survivor trades. The naive comparison is tautological.

## Prediction (the falsifiable form)
The time-exit RULE adds value relative to not having it: a portfolio
with the time exit effectively disabled (positions run to stop/target
only, in practice implemented as a very large max_bars -- see
Implementation note below) has LOWER expectancy than the current
configuration, on TRAIN, consistently across all three instruments --
and the improvement exceeds what the same rule change produces on the
null model.

## Implementation note (amendment before commit)
Arm B is NOT literal removal of the time-exit rule -- some trades would
then never close within the TRAIN/OOS window, which breaks P&L
accounting and comparability. Arm B is implemented as
MAX_BARS_IN_TRADE = 500 (a cap far beyond anything actually reached on
H4 data at this stop/target geometry), which is functionally
equivalent to "no time exit" for every trade that isn't already an
extreme outlier, while keeping every trade closeable and every arm's
output directly comparable.

## Test design
- Data: TRAIN window only (2019-01-01 -> 2022-01-01). OOS stays sealed.
- Arm A: current config (MAX_BARS_IN_TRADE = 10, as shipped).
- Arm B: identical config, MAX_BARS_IN_TRADE = 500.
- Both arms run against the IDENTICAL prepared signal dataframe (same
  entries) -- only the exit rule differs between arms, per instrument.
- Both arms run for: real strategy AND null model (100 seeds each, same
  100 seeds/entry-selections already used in M2 -- not re-drawn).
- Metric: expectancy/trade and PF, A minus B, per instrument, computed
  over TRAIN-window trades only in each arm's own output.
- The claim survives only if (A-B) for the real strategy is positive
  on all three instruments AND exceeds the null's (A-B) distribution
  at the 95th percentile on at least two.
- Secondary (descriptive only, no claim): sweep max_bars in {4, 6, 8,
  12, 16} on TRAIN to characterize sensitivity. No parameter is
  selected from this sweep in this registration.

## "Affected trades" -- definition (amendment before commit)
n >= 30 counts trades whose REALIZED OUTCOME differs between arms, not
total trades run. Because both arms enforce max_open_trades=1, a trade
closing earlier in Arm A (via time exit) can free the position slot for
a subsequent signal that Arm B -- still holding the earlier trade open
-- would skip entirely. This means Arm A and Arm B can have different
total trade COUNTS, not just different exits on the same trades. Given
this, "affected trades" is defined and computed as, per instrument:
  affected = (Arm A TRAIN trades with exit_reason == "time_exit")
           + (Arm A TRAIN trades with no matching entry_dt in Arm B's
              TRAIN trade list, i.e. entries Arm A's faster turnover
              allowed that Arm B's longer-held positions blocked)
This is a conservative count of trades whose outcome or existence
differs because of the rule, not a claim that every one of these trades
individually proves the hypothesis -- it is the effective sample size
against which the n>=30 threshold is checked.

## Kill criteria (any one refutes)
- (A-B) <= 0 on any instrument for the real strategy.
- Real (A-B) inside the null's (A-B) 95% band on two or more
  instruments (the rule helps random entries just as much -> the
  "structure" is mechanical, not strategy-specific).
- Fewer than 30 TRAIN "affected trades" (per the definition above) per
  instrument -> INSUFFICIENT EVIDENCE.
- Effect vanishes under cost +50% stress.

## What I will NOT do
- Compare exit-reason subgroups within one arm (survivorship trap).
- Select a max_bars value from the sensitivity sweep.
- Unseal OOS.
- Re-draw the null model's 100 seeds/entry-selections -- Arm A and Arm B
  for the null must use the SAME 100 seed draws as M2, so the only
  variable changing between arms is the exit rule, not entry selection.
