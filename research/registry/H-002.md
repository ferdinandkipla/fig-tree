### Why this is a clean kill, not a marginal one
- **Best session is different on all three instruments** (tokyo /
  new_york / london) -- the consistency criterion, described in the
  registration as doing "the heavy lifting," fails outright.
- **Worst session inconsistent too** (new_york / london / new_york --
  only 2 of 3 agree).
- **p > 0.05 on two of three instruments** (USDJPY, XAUUSD).
- **The one "significant" result (GBPJPY, p=0.0000) is smaller than
  seed-to-seed noise** -- observed stat 3.66 vs. seed dispersion 13.81.
  This is exactly the pseudoreplication trap the registration's design
  note 2 was written to catch: pooling 100 seeds inflates n into the
  tens of thousands, making even a noise-level effect look
  "statistically significant" in the pooled test, while the effect size
  itself is well within the range of pure entry-timing variation across
  seeds. Reporting GBPJPY's p=0.0000 without this check would have been
  a textbook false positive.

## Reframe note (recorded before test)
Original framing (session-conditioning trend_pullback entries) abandoned
before testing: M2 showed the entry filter is worse than random, so any
session effect inside it would be uninterpretable. This version tests
whether session structure exists in the MARKET, using the null model's
random-entry trades. No trades were examined by session prior to this
registration.

## Prediction
Session-conditioned expectancy in the null model's random long entries
is non-uniform: at least one session's expectancy differs from the
pooled mean by more than chance, with the SAME session ranked
best (and same ranked worst) across all three instruments.

## Test design
- Data: existing M2 null runs (research/null_runs/), 100 seeds per
  instrument, TRAIN window trades only (2019-01-01 -> 2022-01-01).
  OOS-window null trades excluded and unexamined.
- Grouping: session at entry (tokyo / london / new_york) as captured
  in ENTRY_FEATURES.
- Metric: expectancy/trade per session per instrument, pooled across
  seeds; seed-level dispersion reported.
- Significance: permutation null-of-the-null -- shuffle session labels
  within instrument 1,000x, compute max |session mean - pooled mean|;
  observed statistic must exceed the 95th percentile.
- Consistency: best and worst session must be identical across all
  three instruments.

## Kill criteria (any one refutes)
- Permutation p > 0.05 on any instrument.
- Best/worst session ranking not identical across all three.
- Fewer than 30 TRAIN trades in any session bucket per instrument
  (pooled across seeds this should be trivially met; check anyway)
  -> INSUFFICIENT EVIDENCE.
- Effect vanishes under cost +50% stress.

## What I will NOT do
- Sub-slice sessions (e.g., "london open hour") after seeing results.
- Promote a session filter onto trend_pullback (that strategy is dead).
- Unseal OOS (real-strategy or null).

## Design notes (recorded before test)

1. The consistency criterion (identical best/worst session ranking
   across all three instruments) is doing the heavy lifting, not the
   permutation p-value. With only 3 session buckets and pooled n in the
   thousands, permutation significance is easy to hit by chance on one
   instrument alone. Cross-instrument replication is the real bar:
   session effects that don't replicate across JPY pairs and gold are
   not a building block, regardless of any single instrument's p-value.

2. Seed-pseudoreplication caution: pooling 100 seeds inflates n, but
   trades from different seeds on the same instrument can land on
   overlapping bars -- they are not 100 independent draws of the market.
   Seed-level dispersion (per-seed, per-session expectancy) is reported
   alongside the pooled permutation result specifically so the
   permutation p-value can be sanity-checked against between-seed
   noise. If the observed effect is smaller than seed-to-seed variance,
   the p-value is not to be trusted even if it clears 0.05.

## Honest prior on measurement granularity (recorded before test)

Sessions are defined in server time; H4 bars are coarse relative to
session boundaries -- a single 4-hour bar labeled "tokyo" can span
meaningful overlap into adjacent sessions. A null result here may mean
"no session structure at H4 granularity," not "no session structure
exists in the market at all." This distinction is recorded now so it
can be stated plainly in the findings report regardless of which way
the test resolves, rather than adjudicated after the fact.
